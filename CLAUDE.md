# CLAUDE.md

This repository hosts the **documentation site** for the *Tuntun Go Agentic Chatbot* — an enterprise-grade, high-concurrency agentic chatbot built in Go (Gin + LangChainGo) for the financial sector. The site is a Hugo project located under `hugo_docs/`.

> Note: this repo contains only the docs/Hugo site, not the Go application source. The CLAUDE.md describes where the documentation content lives.

## The two important parts

### 1. The main (home) page
`hugo_docs/layouts/page/custom-home.html`

This single template is the project's landing page and the highest-signal overview of the whole system. It contains:
- **System Architecture** — the full Mermaid diagram of the decoupled, two-stage worker pipeline (cache fast-lane → priority queue → follower/leader processing tiers → broadcast).
- **Core Engineering & Features** — feature cards (cache fast-lane, priority dispatch, circuit breakers, single-flight, broadcasting, janitor, RAG, observability) each with an expandable popup explanation defined inline in the page's `<script>`.
- **Available Tools** — the full catalog of LLM tools (market/news, valuation/research, financial reports, financial ratios, `frequently_asked` RAG).
- **API Endpoints** — `/chat/submit`, `/chat/stream/:id`, `/chat/status/:id`, `/chat/cancel/:id`, `/chat/update_reaction`, `/chat/get_chat_history`.
- **Installation & Setup**, **Configuration** (the full annotated `.env`), **Usage & Testing** (every test suite and how to run it), and **Development** (project structure, adding a new tool).

When you need the canonical, up-to-date description of *what the system is and how it behaves*, edit/read this file first.

### 2. The deep-dive documentation
`hugo_docs/content/docs/`

Markdown content rendered into the documentation site. Organized into sections (each has an `_index.md` with the section title/blurb):

- `architectures/` — architectures the chatbot uses (circuit breaker, data-driven tools, design patterns, event-driven janitor, frozen concurrency, jsonless, persistent queue, prestream deadlock, stream broadcaster).
- `caller_insights/` — discussions of the caller tools (concurrent caller, RAG stream, tools).
- `general_go/` — Go patterns used in the chatbot (busy-wait loops, channels/`go_arrow`, goroutines, request locking, event-driven design).
- `manager_insights/` — architectural docs on the Manager module (adapter pattern, broadcast pattern, concurrency vs parallelism, decoupling, race conditions, observability/`sauron`).
- `narratives/` — feature stories told as narratives.
- `python_stuff/` — Python-paradigm discussions (Python vs Go, Python workers).
- `frequently_asked/` — FAQ entries (design, optimization, RAG, semaphore Q&A).

## Working with the Hugo site

The site lives entirely under `hugo_docs/`:
- `layouts/` — templates (the home page is `layouts/page/custom-home.html`).
- `content/docs/` — the documentation markdown (see above).
- `public/` — the generated/built output (do not hand-edit; it is regenerated by Hugo).

To preview or rebuild, run Hugo from the `hugo_docs/` directory (`hugo server` for live preview, `hugo` to build into `public/`).
